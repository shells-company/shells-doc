The HTTP API lets you index files into a dataset and run searches over the
contents. Searches default to `grep`, but the search endpoint also accepts an
arbitrary search string and an optional `command` field so you can swap in any
compatible search tool available on the server.

## Base URL

When running locally, the API is served at `http://localhost:8080`.

## Endpoints

- `POST /api/index` — index a file into a dataset.
- `POST /api/search` — search a dataset with grep (or another configured command).
- `GET /api/{dataset}/tags` — list the tags currently in use for a dataset.
