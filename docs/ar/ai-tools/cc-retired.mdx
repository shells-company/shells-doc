Claude Code هو أداة CLI الرسمية من Anthropic. سيساعدك هذا الدليل على إعداد Claude Code لمساعدتك في كتابة وصيانة التوثيق الخاص بك.

## المتطلبات الأساسية

asdf

- اشتراك Claude نشط (Pro أو Max أو وصول API)
- كيف حالك
- مرحباً يا سيدي
- تغيير
- المزيد من التغييرات

حسناً من فضلك من فضلك من فضلك المزيد من المعلومات

## الإعداد

1. ثبّت Claude Code عالمياً:

```bash
npm install -g @anthropic-ai/claude-code
```

2. انتقل إلى دليل التوثيق.
3. (اختياري) أضف ملف `CLAUDE.md` أدناه إلى مشروعك.
4. شغّل `claude` للبدء.

## إنشاء `CLAUDE.md`

أنشئ ملف `CLAUDE.md` في جذر مستودع التوثيق لتدريب Claude Code على معايير التوثيق الخاصة بك:

```markdown
# توثيق Mintlify

## علاقة العمل
- يمكنك الاعتراض على الأفكار - يمكن أن يؤدي ذلك إلى توثيق أفضل. استشهد بالمصادر واشرح منطقك عند فعل ذلك
- اطلب دائماً التوضيح بدلاً من وضع افتراضات
- لا تكذب أبداً أو تخمّن أو تختلق معلومات

## سياق المشروع
- التنسيق: ملفات MDX مع YAML frontmatter
- الإعداد: docs.json للتنقل والمظهر والإعدادات
- المكوّنات: مكوّنات Mintlify

## استراتيجية المحتوى
- وثّق ما يكفي فقط لنجاح المستخدم - ليس أكثر من اللازم ولا أقل
- أعطِ الأولوية لدقة المعلومات وقابليتها للاستخدام
- اجعل المحتوى مستداماً عند الإمكان
- ابحث عن المعلومات الموجودة قبل إضافة محتوى جديد. تجنّب التكرار إلا لسبب استراتيجي
- راجع الأنماط الحالية للحفاظ على الاتساق
- ابدأ بإجراء أصغر التغييرات المعقولة

## متطلبات الـ Frontmatter للصفحات
- title: عنوان واضح ووصفي للصفحة
- description: ملخص موجز لتحسين محركات البحث/التنقل

## معايير الكتابة
- صيغة المخاطب ("أنت")
- المتطلبات الأساسية في بداية المحتوى الإجرائي
- اختبر جميع أمثلة الكود قبل النشر
- طابق نمط وتنسيق الصفحات الحالية
- أدرج حالات الاستخدام الأساسية والمتقدمة
- وسوم اللغة على جميع كتل الكود
- نص بديل على جميع الصور
- مسارات نسبية للروابط الداخلية

## سير عمل Git
- لا تستخدم أبداً --no-verify عند الـ commit
- اسأل كيفية التعامل مع التغييرات غير المرسلة قبل البدء
- أنشئ فرعاً جديداً عند عدم وجود فرع واضح للتغييرات
- قم بالـ commit بشكل متكرر طوال التطوير
- لا تتجاوز أو تعطّل أبداً خطافات pre-commit

## لا تفعل
- تخطّي frontmatter في أي ملف MDX
- استخدام عناوين URL مطلقة للروابط الداخلية
- تضمين أمثلة كود غير مختبرة
- وضع افتراضات - اطلب التوضيح دائماً
```
