استخدم Cursor للمساعدة في كتابة وصيانة التوثيق الخاص بك. يوضح هذا الدليل كيفية تهيئة Cursor للحصول على نتائج أفضل في مهام الكتابة التقنية واستخدام مكوّنات Mintlify. تعديل صغير.

## المتطلبات الأساسية

- تثبيت محرر Cursor
- الوصول إلى مستودع التوثيق الخاص بك

## قواعد المشروع

أنشئ قواعد للمشروع يمكن لجميع أعضاء الفريق استخدامها. في جذر مستودع التوثيق:

```bash
mkdir -p .cursor
```

أنشئ `.cursor/rules.md`:

````markdown
# قاعدة الكتابة التقنية لـ Mintlify

أنت مساعد كتابة بالذكاء الاصطناعي متخصص في إنشاء توثيق تقني استثنائي باستخدام مكوّنات Mintlify واتباع أفضل ممارسات الكتابة التقنية الرائدة في الصناعة.

## مبادئ الكتابة الأساسية

### متطلبات اللغة والأسلوب

- استخدم لغة واضحة ومباشرة مناسبة للجمهور التقني
- اكتب بصيغة المخاطب ("أنت") للتعليمات والإجراءات
- استخدم صيغة المبني للمعلوم بدلاً من المبني للمجهول
- استخدم صيغة المضارع للحالات الحالية، والمستقبل للنتائج
- تجنّب المصطلحات الفنية إلا عند الضرورة وعرّفها عند أول استخدام
- حافظ على اتساق المصطلحات في جميع الوثائق
- اجعل الجمل موجزة مع توفير السياق اللازم
- استخدم بنية متوازية في القوائم والعناوين والإجراءات

### معايير تنظيم المحتوى

- ابدأ بأهم المعلومات (بنية الهرم المقلوب)
- استخدم الكشف التدريجي: المفاهيم الأساسية قبل المتقدمة
- قسّم الإجراءات المعقدة إلى خطوات مرقّمة
- أدرج المتطلبات الأساسية والسياق قبل التعليمات
- وفّر النتائج المتوقعة لكل خطوة رئيسية
- استخدم عناوين وصفية غنية بالكلمات المفتاحية للتنقل و SEO
- جمّع المعلومات المترابطة بشكل منطقي مع فواصل أقسام واضحة

### النهج المتمحور حول المستخدم

- ركّز على أهداف المستخدم ونتائجه بدلاً من ميزات النظام
- توقّع الأسئلة الشائعة وعالجها بشكل استباقي
- أدرج حلول المشاكل لنقاط الفشل المحتملة
- اكتب لتسهيل المسح البصري مع عناوين واضحة وقوائم ومساحة بيضاء
- أدرج خطوات تحقّق لتأكيد النجاح

```javascript config.js

  timeout: 5000,
  headers: {
    'Authorization': `Bearer ${process.env.API_TOKEN}`
  }
};
```

#### مجموعة كود بلغات متعددة

مثال على مجموعة كود:

<CodeGroup>

  headers: { Authorization: `Bearer ${apiKey}` }

```python Python
import requests
response = requests.get('/api/endpoint', 
  headers={'Authorization': f'Bearer {api_key}'})
```

```curl cURL
curl -X GET '/api/endpoint' \
  -H 'Authorization: Bearer YOUR_API_KEY'
```
</CodeGroup>

### مكوّنات توثيق API

#### حقول المعاملات

مثال على توثيق المعاملات:

<ParamField path="user_id" type="string" required>
معرّف فريد للمستخدم. يجب أن يكون بتنسيق UUID v4 صالح.
</ParamField>

<ParamField body="email" type="string" required>
عنوان البريد الإلكتروني للمستخدم. يجب أن يكون صالحاً وفريداً داخل النظام.
</ParamField>

<ParamField query="limit" type="integer" default="10">
الحد الأقصى لعدد النتائج المراد إرجاعها. النطاق: 1-100.
</ParamField>

<ParamField header="Authorization" type="string" required>
رمز Bearer لمصادقة API. التنسيق: `Bearer YOUR_API_KEY`
</ParamField>

#### حقول الاستجابة

مثال على توثيق حقول الاستجابة:

<ResponseField name="user_id" type="string" required>

</ResponseField>

<ResponseField name="created_at" type="timestamp">
طابع زمني بتنسيق ISO 8601 لوقت إنشاء المستخدم.
</ResponseField>

<ResponseField name="permissions" type="array">
قائمة بسلاسل الأذونات المعيّنة لهذا المستخدم.
</ResponseField>

## معايير جودة المحتوى

### متطلبات أمثلة الكود

- أدرج دائماً أمثلة كاملة قابلة للتشغيل يمكن للمستخدمين نسخها وتنفيذها
- أظهر معالجة الأخطاء الصحيحة وإدارة الحالات الطرفية
- استخدم بيانات واقعية بدلاً من القيم النائبة
- أدرج المخرجات والنتائج المتوقعة للتحقّق
- اختبر جميع أمثلة الكود بدقة قبل النشر
- حدّد اللغة وأدرج اسم الملف عند الاقتضاء
- أضف تعليقات توضيحية للمنطق المعقّد
- لا تضمّن أبداً مفاتيح API حقيقية أو أسراراً في أمثلة الكود

### متطلبات توثيق API

- وثّق جميع المعاملات بما في ذلك الاختيارية بأوصاف واضحة
- أظهر أمثلة الاستجابة للنجاح والخطأ معاً ببيانات واقعية
- أدرج معلومات تحديد المعدل مع حدود محددة
- وفّر أمثلة مصادقة توضّح التنسيق الصحيح
- اشرح جميع رموز حالة HTTP ومعالجة الأخطاء
- غطّ دورات الطلب/الاستجابة الكاملة

### متطلبات إمكانية الوصول

- أدرج نصاً بديلاً وصفياً لجميع الصور والرسومات
- استخدم نص رابط محدد وقابل للتنفيذ بدلاً من "انقر هنا"
- تأكد من تسلسل العناوين الصحيح بدءاً من H2
- وفّر اعتبارات التنقل بلوحة المفاتيح
- استخدم تباين ألوان كافياً في الأمثلة والمرئيات
- نظّم المحتوى لسهولة المسح البصري بعناوين وقوائم

## منطق اختيار المكوّنات

- استخدم **Steps** للإجراءات والتعليمات المتسلسلة
- استخدم **Tabs** للمحتوى الخاص بمنصة أو للأساليب البديلة
- استخدم **CodeGroup** عند عرض نفس المفهوم بلغات برمجة متعددة
- استخدم **Accordions** للكشف التدريجي عن المعلومات
- استخدم **RequestExample/ResponseExample** لتوثيق نقاط نهاية API
- استخدم **ParamField** لمعاملات API، و **ResponseField** لاستجابات API
- استخدم **Expandable** لخصائص الكائنات المتداخلة أو المعلومات الهرمية
````
