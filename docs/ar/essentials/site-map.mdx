قائمة التنقّل هي قائمة الروابط الظاهرة على كل موقع.

من المرجّح أن تُحدّث `docs.json` في كل مرة تضيف فيها صفحة جديدة. الصفحات لا تظهر تلقائيًا.

## صيغة التنقّل

صيغة التنقّل لدينا تكراريّة، مما يعني أنه يمكنك إنشاء مجموعات تنقّل متداخلة. لا تحتاج إلى تضمين `.mdx` في أسماء الصفحات.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## المجلدات

ضع ببساطة ملفات MDX في مجلدات وحدِّث المسارات في `docs.json`.

على سبيل المثال، للحصول على صفحة عند `https://yoursite.com/your-folder/your-page` عليك إنشاء مجلد يُسمى `your-folder` يحتوي على ملف MDX يُسمى `your-page.mdx`.

<Warning>
  لا يمكنك استخدام `api` كاسم لمجلد ما لم تُداخِله داخل مجلد آخر. يستخدم Mintlify إطار Next.js الذي يحجز مجلد `api` من المستوى الأعلى لاستدعاءات الخادم الداخلية. سيتم قبول اسم مجلد مثل `api-reference`.
</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## الصفحات المخفية

ملفات MDX غير المضمَّنة في `docs.json` لن تظهر في الشريط الجانبي ولكنها متاحة عبر شريط البحث ومن خلال الربط بها مباشرةً.
